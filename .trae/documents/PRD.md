## 1. 产品概述

戏曲传承人口齿录管理系统是面向省非遗保护中心工作人员的全栈 Web 应用，用于集中管理全省二十多个地方剧种的传承人名录、代表剧目、演出团体及音视频资料，替代散落在各 Excel 表的低效查询方式。

- 主要目的：将分散的戏曲传承人数据统一入库，提供多维检索、详情查阅、后台维护与剧种科普的一体化平台。
- 解决问题：多表查询费劲、音视频资料难归档、剧种知识难聚合、移动端访问缺失。
- 目标用户：非遗保护中心工作人员（管理员）、文化研究人员与公众（访客）。系统兼具文化典藏的严谨性与现代化检索体验。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通访客 | 无需注册 | 浏览名录查询页、传承人详情、剧种百科，支持拼音首字母索引与模糊搜索 |
| 管理员 | 后台分配账号 | 全部访客权限 + 传承人信息增删改查、批量导入 Excel、头像/音视频文件上传 |

### 2.2 功能模块

1. **名录查询页**：多维筛选条（剧种 / 行当 / 地区 / 师承关系 / 年龄段）、姓氏拼音首字母快速索引、传承人卡片网格、模糊搜索框、详情抽屉。
2. **传承人详情页**：基本信息、学艺经历时间线、代表剧目、获奖情况、录音录像资料列表（可在线播放）。
3. **后台管理页**：传承人列表表格、新增/编辑表单、删除确认、Excel 批量导入、头像与音视频文件本地上传。
4. **剧种百科页**：剧种介绍（历史渊源 / 艺术特点 / 经典剧目 / 主要流派 / 分布地区）、图文展示、关联活跃传承人卡片、关联现存剧团列表。

## 3. 核心流程

**访客查询流程**：访客进入名录查询页 → 通过筛选条/拼音索引/模糊搜索定位传承人 → 点击卡片打开详情抽屉 → 查看档案与音视频资料。

**管理员导入流程**：管理员登录后台 → 上传 Excel 文件 → 后端解析校验 → 批量写入数据库 → 返回成功/失败条数 → 在列表中核对数据。

**剧种浏览流程**：访客进入剧种百科页 → 浏览剧种卡片墙 → 进入剧种详情 → 阅读历史/艺术特点 → 查看关联传承人与剧团 → 跳转传承人详情。

## 4. 用户界面设计

### 4.1 设计风格

**美学方向：新中式典藏（Neo-Chinese Heritage / 编辑式典藏）**——以传统戏曲文化为底蕴，用现代编辑排版重新诠释，严谨而不失温度。

- 主色：墨色 `#1A1714`（文字主色）、宣纸米 `#F5F1E8`（背景）、朱砂红 `#9E2B25`（强调/印章）、鎏金 `#B8893B`（点缀/分割）。
- 辅助色：青瓷 `#5C7A6B`（次要标签）、淡墨 `#6B6258`（次要文字）。
- 按钮风格：主按钮朱砂红实底带微凹投影；次按钮宣纸描边；强调按钮可带印章式方角。
- 字体：标题用衬线中文展示字体 `Noto Serif SC`（典藏感）；正文用 `LXGW WenKai` 或 `Noto Sans SC`（清晰易读）；印章/编号可用 `Ma Shan Zheng` 点缀。
- 布局风格：编辑式杂志排版，强字号层级，大量留白，不对称网格，分章节编号；卡片以宣纸纹理 + 细墨边描边。
- 图标/装饰：印章式角标、水墨晕染渐变背景、竖排标题装饰线、页眉红色印章徽记。

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|

| 传承人详情页 | 档案头 | 大号衬线姓名、剧种行当标签、朱砂分隔线 |

### 4.3 响应式

桌面优先，移动端自适应：查询页卡片网格由 4 列降为 1 列；筛选条折叠为下拉；详情抽屉移动端全屏；拼音条移动端可横向滚动；后台表格移动端转为卡片列表。触摸目标 ≥ 44px，核心操作无横向滚动阻碍。

### 4.4 装饰细节

- 背景叠加宣纸噪点纹理与极淡水墨渐变，营造典藏氛围。
- 卡片右上角朱砂方印作为传承人/剧种编号徽记。
- 章节标题以竖排编号 + 横线装饰，呼应古籍版式。
- 悬浮态使用墨色描边上移 + 柔和投影的微交互。
